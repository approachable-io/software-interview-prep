What is a typical use case for anonymous functions in Javascript?

Anonymous functions can be used for

  1. Brevity
  2. Isolating scope

Brevity

If you are only using a function once and you're calling it right away, you probably don't want to wettten up your code by defining it. You could write colde like this:

var habits = ['making eye contact uncomfortably long',
  'instigating slouching contests',
  'saying good morning cheerfully to people who are probably not morning people'];
function habbitsCallback(habbit, i) {
  console.log(i + '. ' + habbit + '.');
);

habbits.forEach(habbitsCallback);

But usually you will not define a callback function if you're only using it once. You'd use an anonymous function instead:

habits.forEach(function(habbit, i) {
  console.log(i + '. ' + habbit + '.');
});

Isolating scope

Sometimes you just want a little privacy. If you are writing code in a section that could possibly conflict with variable names or something from another section of your code, you can isolate it with an anonymous function. One example is with an immediatly involked function expression:

(function() {
  var habits = 'I have some interesting habits.';
}();
console.log(habits); //habits is not defined;

I have never seen that used, so to me a more familiar example is when using a Javascript library like jQuery or framework like Angular.js:

app.controller('HabbitController', ['HabbitService', function (HabbitService) {
  var habbit = 'Eating cold pop tarts';
}
console.log(habbit); //habbit is not defined.

More detailed discussion of local and global scope with anonymous functions: https://stackoverflow.com/questions/17058606/why-using-self-executing-function
